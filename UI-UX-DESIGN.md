# 设计总纲

## 设计哲学

```
留白不是浪费，是让内容呼吸。

每个屏幕只说一件事。

用户的拇指决定了交互的位置。

知识不是平铺的，是长在树上的。
```

## 全局规范

### 触控规范

```
┌─────────────────────────┐
│                         │
│     ╔═══════════════╗   │  ← 自然触达区（最轻松）
│     ║  拇指自然区域  ║   │     主要操作放这里
│     ║               ║   │
│     ╚═══════════════╝   │
│     ┌───────────────┐   │  ← 伸展触达区（需伸手）
│     │  拇指伸展区域  │   │     次要操作放这里
│     └───────────────┘   │
│     ················    │  ← 困难触达区（需换手持握）
│                         │     不放任何核心操作
└─────────────────────────┘
```

| 区域 | 位置 | 放什么 |
|------|------|--------|
| 自然触达 | 屏幕下半部分 | 翻转按钮、自评按钮（核心交互） |
| 伸展触达 | 屏幕中间 | 卡片内容、进度条 |
| 困难触达 | 屏幕顶部 | 标题、返回按钮（低频操作） |

### 间距系统（4pt 基准）

```
单位：pt（1pt = 1/72 inch）

  4pt  — 最小间距（图标与文字、标签之间）
  8pt  — 紧凑间距（卡片内元素）
 12pt  — 默认内边距（卡片 padding）
 16pt  — 标准间距（区块之间、列表项之间）
 20pt  — 宽松间距（页面左右 margin）
 24pt  — 章节间距
 32pt  — 大留白（页面顶部、区块首尾）
 48pt  — 超大留白（完成画面上下）
```

### 字体规范

```
标题（页面级）：   SF Pro Display / 28pt / Bold
标题（区块级）：   SF Pro Display / 20pt / Semibold
标题（卡片级）：   SF Pro Display / 17pt / Medium
正文：            SF Pro Text / 15pt / Regular
辅助信息：        SF Pro Text / 13pt / Regular
标签/角标：       SF Pro Text / 11pt / Medium
按钮文字：        SF Pro Text / 17pt / Semibold

行高：1.5（正文）/ 1.3（标题）
字距：-0.02em（标题）/ 0（正文）
```

> 卡片是绝对主角。没有多余元素干扰注意力。用户只需关注「问题→回忆→翻转→自评」这一个流。

## 视觉层级

```
第 1 级（绝对焦点）：  卡片（占据屏幕 60%+）
第 2 级（行动）：      翻转按钮 / 自评按钮
第 3 级（辅助信息）：  进度条 + 当前序号
第 4 级（退出路径）：  返回按钮

## 卡片尺寸精确计算

```

卡片内部 padding：20pt（四边）
可用内容宽度：335 - 40 = 295pt

卡片高度：
  上 padding        20pt

  问题文字区          ~120pt（4 行 × 30pt）

## 分类标签设计

知识树面包屑是学习页的关键差异化设计——它告诉用户「你正在知识树的哪个位置学习」。

```
┌──────────────────────────────────┐
│  习近平新时代 › 核心要义 › "八个明确"  │
│  ──────────────────────────────── │
│  这是一个可点击的路径，不是装饰。     │
│  点击任意层级 → 返回该层级的学习列表。│
└──────────────────────────────────┘
```

视觉样式：
- 字号：11pt
- 颜色：#AEAEB2（三级文字）
- 层级分隔符：`›`（非 `>` ，更精致）
- 当前层级（最后一级）颜色加深为 #6E6E73
- 整个面包屑左对齐，位于卡片内容区顶部

| 面包屑 | 点击任意层级 | 弹出该层级的知识点列表 Sheet |

```

```

# 页面流与转场

## 转场动画规范

```
页面 push（进入子页面）：
  新页面从右滑入（300ms ease-out）
  旧页面向左移动 30%（视差效果）

页面 pop（返回上级）：
  当前页面向右滑出（300ms ease-out）
  下级页面从左侧 30% 移到原位

Tab 切换：
  无滑动动画，直接淡入淡出（200ms）
  因为 Tab 之间是并行关系，不是层级关系

模态出现（Sheet / Dialog）：
  从底部滑入（400ms spring）
  背景变暗（200ms）

模态消失：
  滑出到底部（300ms）
  背景变亮（200ms）
```

# 知识树呈现设计（关键差异化）

## 设计原则

> 知识不能平铺。用户每次学习都必须感知到「这个知识点在整体知识体系中的位置」。

## 三处呈现知识树

### 1. 卡片面包屑（学习/复习页）

```
  习近平新时代 › 核心要义 › "八个明确" › 第一条
  ─────────────────────────────────────────────
```

这是最轻量的呈现方式。一行文字，告诉用户当前所在位置。

### 2. 知识领域树（首页 + 知识图谱）

```
  习近平新时代中国特色社会主义思想           62%
  ├── 核心要义                     ● 58%
  │   ├── 坚持和发展中国特色社会主义  ● 91%
  │   ├── 实现中华民族伟大复兴      ● 45%
  │   └── 以人民为中心              ● 62%
  ├── "八个明确"                   ● 70%
  │   ├── 明确一                    ● 85%
  │   ├── 明确二                    ● 60%
  │   └── ...
  └── "十四个坚持"                 ● 55%
      ├── 坚持党对一切工作的领导    ● 92%
      └── ...
```

首页只展示到第二层（领域→子领域）。更深层级在知识图谱中呈现。

### 3. 知识图谱（重点强化页）

```
                 ┌──────────────────────────────┐
                 │        知识图谱全貌           │
                 │                              │
                 │       [习思想]  ────┐         │
                 │       ╱    ╲       │         │
                 │   [八明确] [十四坚持]         │
                 │    ╱  ╲      ╱  ╲           │
                 │ [明1][明2] [坚1][坚2]         │
                 │    ╲  ╱      ╲  ╱           │
                 │   [关联关系]  [关联关系]       │
                 │                              │
                 └──────────────────────────────┘
```

## 知识树节点视觉语言

# 加载与空状态

## 加载状态

```
骨架屏（首页）：
  ┌──────────────────────────────────┐
  │          🏛 理论记忆              │
  │                                  │
  │  ┌────────────────────────────┐  │
  │  │       ████████░░░          │  │  ← 灰色占位块
  │  │       ██████               │  │     带 shimmer 动画
  │  └────────────────────────────┘  │
  │                                  │
  │  ████████████░░░                 │
  │  ████████████░░░                 │
  └──────────────────────────────────┘

AI 提炼中（导入页）：
  ┌──────────────────────────────────┐
  │                                  │
  │          ◌                       │  ← 旋转动画
  │                                  │
  │      AI 正在提炼知识点…          │
  │                                  │
  │  第 1 轮：提取知识点  ✓          │  ← 实时进度
  │  第 2 轮：逐句比对…  进行中     │
  │  第 3 轮：合并去重…  等待       │
  │                                  │
  │  这可能需要 30-60 秒             │
  └──────────────────────────────────┘
```

## 空状态设计原则

- 不显示空白页面。每个空状态都有引导文案 + 行动按钮。
- 图标用灰色，不用彩色（空状态不应该吸引注意力）。
- 文字简短友好，不使用技术术语。

# 手势冲突解决

| 冲突 | 解决 |
|------|------|
| 卡片页纵向滑动 vs 页面返回手势 | 卡片页禁用 iOS 边缘滑动返回（全屏沉浸） |
| 知识图谱缩放 vs 页面滚动 | 图谱区域截获双指手势，单指手势留给页面滚动 |
| 易混概念横向滑动 vs 页面返回 | 概念卡片区域截获横向滑动手势 |
