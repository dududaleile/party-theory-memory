## 页面 1：首页（/）

### 1.1 页面目的

作为用户打开应用看到的第一个画面，首页只做三件事：
- **告诉用户现在该做什么**（有待复习就立即复习）
- **展示学习状态概览**（让用户感知进度）
- **提供一个唯一的行动入口**（开始学习）

### 1.2 布局结构

### 1.3 功能详述

#### 1.3.1 状态卡片组

| 数据项 | 数据来源 | 计算逻辑 |
|--------|---------|---------|
| 今日待复习 | IndexedDB reviewQueue | `count(cards where nextReview <= now)` |
| 持续学习 | IndexedDB reviewLogs | `count(distinct days with review activity)` |
| 记忆保持率 | IndexedDB reviewLogs | `count(quality >= 3 in last 30 reviews) / 30` |

**交互行为：**
- 点击「今日待复习」→ 跳转复习页，筛选到期卡片
- 点击「持续学习」→ 跳转设置页的学习统计（V2 功能）
- 点击「记忆保持率」→ 无跳转，仅信息展示
- 状态数据在页面 onFocus 时刷新

#### 1.3.2 「开始学习」按钮——智能分流逻辑

```
用户点击「开始学习」
  │
  ├─ 有待复习的到期卡片？
  │   YES → 进入复习模式（艾宾浩斯优先）
  │         携带参数：review
  │
  └─ 无到期卡片？
      ├─ 有待学习的新卡片？
      │   YES → 进入学习模式
      │         携带参数：learn
      │
      └─ 新卡片也没有？
          └─ 进入重点强化模式
              提示：「暂无新内容，建议回顾薄弱知识点」
```

**设计理由：** 用户不需要思考「我现在该学还是该复习」——系统替用户做决策。这消除了选择焦虑，降低使用门槛。

#### 1.3.3 知识领域列表

| 属性 | 说明 |
|------|------|
| 排序 | 按 domain.order 字段排序（用户可在设置页拖拽调整，V2） |
| 每项显示 | 领域名称 + 环形进度图 + 百分比 |
| 进度计算 | `该领域 masteredCards / 该领域 totalCards` |
| mastered 定义 | 卡片 ease >= 2.5 且 repetitions >= 3 |
| 点击行为 | 进入该领域的专属学习模式（仅学习该领域的卡片） |
| 空状态 | 「暂无知识领域，请先在设置页导入知识」+ 快捷跳转链接 |

#### 1.3.4 底部 Tab Bar

| 位置 | 图标 | 文字 | 路由 | 角标 |
|------|------|------|------|------|
| 1 | 🏠 | 首页 | / | - |
| 2 | 📖 | 学习 | /learn | 新卡片数量 |
| 3 | 🔄 | 复习 | /review | 到期卡片数量（红色） |
| 4 | 🎯 | 强化 | /focus | 薄弱点数量（橙色） |
| 5 | ⚙️ | 设置 | /settings | - |

**角标规则：**
- 红色角标 = 需要立即行动（到期复习）
- 橙色角标 = 建议关注（薄弱点）
- 灰色角标 = 信息提示（新卡片）
- 角标数字 > 99 显示 "99+"

**Tab Bar 行为：**
- 始终固定在底部，高度 56px
- 卡片全屏学习时隐藏（让出全部屏幕空间给卡片）
- 使用 Framer Motion `layout` 动画平滑过渡
- iOS Safari 安全区域适配：`padding-bottom: env(safe-area-inset-bottom)`

## 页面 2：学习页（/learn）

### 2.1 页面目的

学习新知识点。用户第一次接触某个知识点时，通过「主动回忆 → 翻转 → 自评」完成首次编码。

### 2.2 核心机制：主动回忆

```
传统学习：
  看问题 → 看答案 → "嗯，懂了" → 遗忘

本系统：
  看问题 → 【强制停顿】→ 脑中回忆 → 点击翻转 → 看答案对比 → 自评 → 进入间隔算法
```

**强制停顿设计：**
- 卡片正面只显示问题
- 翻转按钮文案：「我已思考，查看答案」
- 不设倒计时（不给压力）
- 翻转是主动行为，不是自动播放

### 2.4 功能详述

#### 2.4.1 卡片翻转交互

| 属性 | 说明 |
|------|------|
| 动画 | Framer Motion `rotateY` 3D 翻转，持续 400ms |
| 触发 | 仅「我已思考，查看答案」按钮触发，不支持滑动手势（避免误触） |
| 翻转后 | 答案面显示，自评按钮出现（带 fadeIn 动画） |
| 翻转状态 | 不可逆——看到答案后不能「再想想」，必须评分 |
| 黑暗场景 | 卡片颜色柔和，不刺眼（`#1a1a2e` 背景，`#f0f0f0` 文字） |

#### 2.4.2 自评按钮（4 级评分）

| 评级 | 含义 | SM-2 Quality | 颜色 | 后续处理 |
|------|------|-------------|------|---------|
| 完全忘记 | 答案完全没印象 | 0 | 红色 | 重置间隔，今天会再次出现 |
| 模糊 | 记得一些但不确切 | 2 | 橙色 | 间隔缩短 50% |
| 基本记住 | 大部分正确，少量偏差 | 4 | 蓝色 | 正常间隔递进 |
| 完全掌握 | 回忆完全准确 | 5 | 绿色 | 间隔取最大值 |

**设计细节：**
- 4 个按钮等宽均分，无默认选中
- 点击后 300ms 内禁用（防连点）
- 点击任意评级 → 卡片退出动画（向下滑出）→ 自动加载下一张
- 无「跳过」选项——每一张卡片必须评分（这是记忆系统的基本要求）

#### 2.4.3 学习队列管理

| 概念 | 说明 |
|------|------|
| 学习来源 | 从 IndexedDB 获取 `nextReview = null` 的卡片（从未学过） |
| 排序 | 按知识领域层级排序，先学上层概念再学子概念 |
| 每日上限 | 默认 10 张新卡片/天（可在设置调整） |
| 上限到达 | 提示「今日新卡片已达上限（10/10），剩余卡片明日解锁」 |
| 学习完成 | 显示完成画面 + 统计数据 + 进入复习/返回首页选项 |

| 状态 | 处理 |
|------|------|
| 无新卡片 | 显示「暂无新卡片。所有知识点已加入复习队列。你可以：① 回顾薄弱知识点 ② 导入新知识」+ 快捷跳转按钮 |

| 离线 | 完全可用，所有数据在本地 |

## 页面 3：复习页（/review）

### 3.1 页面目的

按照艾宾浩斯遗忘曲线，在最佳时间点复习即将遗忘的知识点。这是整个系统最关键的页面——**记忆留存的核心战场**。

### 3.2 复习模式

系统支持两种复习模式：

**模式 A：自由输入判分模式（论述题/简答题）**

用户输入自己的答案，AI 对比标准答案判分。适用于需要精确措辞的知识点。

**模式 B：卡片翻转自评模式（选择题/概念题）**

与学习页相同的卡片翻转交互。适用于概念记忆类知识点。

**默认行为：**
- 系统默认为所有卡片使用模式 B（自评）
- 标记为「论述题」类型的卡片使用模式 A（输入判分）
- 用户可在知识导入时标记题型

### 3.5 功能详述

#### 3.5.1 复习队列

| 属性 | 说明 |
|------|------|
| 入队条件 | `card.nextReview <= now()` |
| 排序 | 按 `nextReview` 升序（最紧急的排最前） |
| 队列刷新 | 每次进入页面、每次评分后重新计算 |
| 每日上限 | 默认 50 张/天（可在设置调整），超出则提示「今日复习已达上限，剩余 {n} 张明日继续」 |

#### 3.5.2 复习卡片信息

学习页没有但复习页有的信息：

| 信息 | 位置 | 目的 |
|------|------|------|
| 上次复习时间 | 卡片正面底部 | 让用户感知时间间隔 |
| 历史正确率 | 卡片正面底部 | 让用户知道这张卡片的历史表现 |
| 复习次数 | 卡片正面底部 | 让用户看到自己在这张卡片上的投入 |
| 间隔提示 | 翻转后显示 | 「下次复习：{n} 天后」 |

#### 3.5.3 AI 智能判分流程

```
用户提交答案
  │
  ▼
服务端：AI 判分
  │  Prompt 结构：
  │  - 标准答案（满分参考答案）
  │  - 关键要点列表（必须覆盖的点）
  │  - 用户答案
  │
  ▼
返回 JSON：
  {
    score: 0-100,           // 总分
    covered: [...],         // 已覆盖要点
    missing: [...],         // 遗漏要点
    suggestion: "..."       // 简短建议（1句话）
  }
  │
  ▼
客户端展示判分结果卡片
  │
  ▼
用户查看后仍需自评（自评结果用于调节间隔）
```

**判分设计原则：**
- AI 判分是**辅助参考**，不是最终裁判

- 连续 3 次 AI 判分 < 70 → 自动标记为薄弱知识点

### 4.3 功能详述

#### 4.3.1 易混概念识别

**触发机制：**
- 手动触发：用户在设置页点击「扫描易混概念」，调用 AI 分析
- 自动触发：每当知识库新增 ≥ 20 张卡片，系统自动提示是否需要扫描
- AI 输入：所有知识点的 question + answer + keywords
- AI 输出：易混概念对 + 相似点 + 区分要点

**对比卡片设计：**
- 两个概念左右并排（移动端上下排列）
- 差异点用对比色高亮
- AI 生成的「区分要点」放在底部醒目位置
- 支持展开/收起

**数量控制：**
- 最多展示 5 组易混概念（再多会增加认知负荷）
- 按混淆程度排序（AI 输出的 confidence 降序）

#### 4.3.2 薄弱知识点

**标记规则：**

| 条件 | 标记为 |
|------|--------|
| 连续 3 次自评为「完全忘记」或「模糊」 | 薄弱（高优先级） |
| AI 判分连续 3 次 < 70 | 薄弱（高优先级） |
| 最近 5 次中有 3 次错误 | 薄弱（中优先级） |
| easeFactor < 1.5 | 薄弱（低优先级） |

**薄弱点列表：**
- 按错误率降序排列
- 每项显示：知识点问题、错误率、错误次数
- 「强化」按钮进入该知识点的专项复习：
  - 先展示标准答案
  - 再展示 AI 生成的解释/记忆技巧（如果配置了）
  - 最后进入该卡片的间隔重复

**薄弱点移除：**
- 连续 3 次自评为「基本记住」或「完全掌握」→ 自动移除薄弱标记
- 用户手动标记「已掌握」→ 立即移除

#### 4.3.3 论述骨架

**生成逻辑：**
- 用户在知识导入时标记某些知识点为「论述题考点」
- 系统定期（或手动触发）调用 AI 生成论述骨架
- AI Prompt 包含：题目 + 标准答案 + 相关知识点 + 关键词

**骨架结构：**

```
题目：论全面从严治党

【总论点】（一句话核心观点）
全面从严治党是党自我革命的生动实践，
是新时代党的建设的鲜明主题。

【分论点】（3-4 个，逻辑递进）
1. 政治建设是根本——决定党的建设方向和效果
   关键词：政治方向、政治立场、政治原则
   
2. 思想建设是基础——补足精神之钙
   关键词：理想信念、理论武装、初心使命
   
3. 组织建设是保障——建强战斗堡垒
   关键词：基层组织、干部队伍、人才强国
   
4. 作风建设是关键——密切党群关系
   关键词：八项规定、反四风、群众路线

【结论】（回扣总论点 + 升华）
全面从严治党永远在路上，必须常抓不懈、久久为功。

【关键术语】
政治建设、思想建设、组织建设、作风建设、
纪律建设、制度建设、自我革命
```

**展示方式：**
- 卡片列表，每张显示题目 + 总论点摘要
- 点击展开完整骨架
- 支持一键复制（方便粘贴到笔记）
- 骨架中的关键词与知识卡片自动链接

#### 4.3.4 知识图谱

**技术选型：**
- D3.js force simulation（力导向图）
- 渲染在 `<canvas>` 上（性能优于 SVG 当节点 > 50）
- 虚拟化节点（视口外的节点不渲染）

**视觉编码：**

| 视觉属性 | 编码内容 |
|----------|---------|
| 节点大小 | 掌握度（大 = 掌握好） |
| 节点颜色 | 所属知识领域 |
| 节点边框 | 红色边框 = 薄弱点，绿色边框 = 已掌握 |
| 连线粗细 | 关系强度 |
| 连线样式 | 实线 = 父子关系，虚线 = 关联关系 |

**交互：**
- 单指拖拽：平移画布
- 双指捏合：缩放（最小 0.3x，最大 3x）
- 单击节点：高亮该节点及其直接关联节点 + 显示节点详情浮层
- 双击节点：跳转到该知识点的学习卡片
- 初始视图：适配屏幕宽度，自动居中

**性能：**
- 节点数 > 100 时自动降低帧率到 30fps
- 力模拟在闲置 3 秒后自动停止（alpha < 0.01）

### 4.4 页面组织原则

四个区块按优先级垂直排列：
1. **易混概念** → 直接影响考试得分（选择题常见陷阱）
2. **薄弱知识点** → 数据驱动，精准补漏
3. **论述骨架** → 考试输出，大题准备
4. **知识图谱** → 全局理解，知识体系化

用户可根据需要折叠/展开每个区块。折叠状态会被记住。

#### 5.3.2 知识导入——AI 提炼流程

这是整个系统最复杂的 AI 交互，必须多轮校验。

**方式 1：粘贴文本**

```

  │
  ▼
──────────────────────────────────────
Round 1：初次提取
──────────────────────────────────────
  AI Prompt：
    「从以下文本中提取所有理论知识知识点。
      每个知识点包含：
      - 所属领域（推断层级结构）
      - 知识点标题（简练准确）
      - 问题表述（适合作为记忆卡片正面）
      - 标准答案（完整准确，保留原文关键措辞）
      - 关键词（3-5 个）
      - 难度预估（1-5）
      - 题型标记（概念题 / 论述题 / 辨析题）
      
      必须以 JSON 数组返回。」
      
  AI 输出 → { points: [...], domainTree: {...} }
  │
  ▼
──────────────────────────────────────
Round 2：逐句比对遗漏检查
──────────────────────────────────────
  AI Prompt：
    「以下是原文（逐句编号）。
      以下是已提取的知识点列表。
      
      请逐句检查原文，列出所有在知识点中【未覆盖】的内容。
      对于每一处遗漏，说明：
      - 原文句子编号
      - 遗漏内容
      - 建议如何整合（新增知识点 / 补充到已有知识点）
      
      宁可多报，不可漏报。」
      
  AI 输出 → { missedItems: [...] }
  │
  ▼
──────────────────────────────────────
Round 3：合并 + 去重 + 修正
──────────────────────────────────────
  AI Prompt：
    「现有知识点列表 + 遗漏项列表。
      请合并去重，确保：
      1. 无重复知识点
      2. 所有遗漏项已整合
      3. 答案准确无误
      4. 层级结构合理

  AI 输出 → 最终 JSON
  │
  ▼
客户端：
  1. 展示提取结果预览（知识点列表 + 知识树）
  2. 用户可手动增删改（每个字段可编辑）
  3. 用户确认后写入 IndexedDB
  4. 自动生成记忆卡片
```

## 3.2 六个阶段详解

### 阶段 ①：输入——知识获取

**用户操作：**
1. 获取党课学习材料（教材、讲义、笔记、考试大纲）
2. 打开设置页 → 粘贴文本或导入文件

**系统行为：**

编码的目标是将「线性文本」转化为「结构化知识」：

```
原文（线性）：
  "习近平新时代中国特色社会主义思想是全党全国人民

        ↓ AI 编码 ↓

知识树（结构化）：
  习近平新时代中国特色社会主义思想
  ├── 定位
  │   └── 全党全国人民为实现中华民族伟大复兴而奋斗的行动指南
  ├── 核心内容
  │   ├── "八个明确"

  │   │   └── ...

  │       ├── 坚持 1：坚持党对一切工作的领导
  │       └── ...
  └── ...

记忆卡片（每个叶子节点生成 1 张卡片）：
  {
```

**结构化带来的好处：**
- 知识不是孤立的——每次学习一张卡片，用户都能看到它在知识树中的位置
- 学习顺序有逻辑——先学根节点概念，再学叶子节点细节
- 混淆检测有基础——同一父节点下的兄弟节点天然容易混淆，优先检测

### 阶段 ③：存储——间隔重复

**核心：SM-2 改进算法 × 艾宾浩斯遗忘曲线**

```
算法参数（每张卡片独立）：

  easeFactor:   初始 2.5（范围 1.3 ~ ∞）
  interval:      当前间隔（天）
  repetitions:  成功连续次数
  nextReview:   下次复习时间戳
  history:      历史评分记录 [{date, quality, timeSpent}]

评分 → 间隔计算：

  quality = 用户自评（0-5）
  
  if quality >= 3:  // 回忆成功
    if repetitions == 0:
      interval = 1   // 首次成功 = 1 天
    elif repetitions == 1:
      interval = 3   // 第二次 = 3 天
    else:
      interval = interval * easeFactor
    repetitions += 1
  else:  // 回忆失败
    repetitions = 0
    interval = 1    // 重置为 1 天
    // easeFactor 不变（但后续会因低评分降低）
  
  // 更新 easeFactor
  easeFactor = easeFactor + (0.1 - (5 - quality) * (0.08 + (5 - quality) * 0.02))
  if easeFactor < 1.3: easeFactor = 1.3
  
  // 首次学习特殊处理（3 次短间隔）
  if card is NEW:
    第一次评分后 → 5 分钟后复习
    第二次评分后 → 30 分钟后复习
    第三次评分后 → 12 小时后复习
    第四次开始 → 正常的每日间隔
  
  nextReview = now + interval * 86400000  // 毫秒
```

**遗忘曲线动态调节：**

如果一张卡片在复习时被评「完全忘记」，说明该卡片的遗忘速度快于预期。系统会自动：
- 重置间隔为 1 天
- 降低 easeFactor（最低 1.3）
- 增加该卡片在薄弱点列表中的权重

如果一张卡片连续 3 次评「完全掌握」，说明记忆足够牢固：
- easeFactor 快速增长
- 间隔加速延长（7d → 15d → 30d → 60d → 120d）

**实际复习节奏示意：**

```
Day 1    学习新卡片（10 张）
          5min 后复习 | 30min 后复习 | 12h 后复习
Day 2    复习（1 天间隔）+ 学习新卡片
Day 4    复习（3 天间隔）
Day 8    复习（7 天间隔）
Day 16   复习（15 天间隔）
Day 31   复习（30 天间隔）
...
```

### 阶段 ④：提取——主动回忆 + 自评

这是用户每天实际体验的交互：

```
1. 看到问题面
2. 脑中回忆（强制停顿）
3. 点击翻转
4. 查看答案，对比自己的回忆
5. 自评（0-5）

```

**为什么这个流程有效：**

| 认知科学原理 | 如何体现在系统中 |
|------------|----------------|
| **生成效应** | 翻转前必须自己回忆，不是被动阅读 |
| **测试效应** | 每次复习都是一次小测试 |
| **间隔效应** | 复习在遗忘临界点进行，不是随意重复 |
| **难度适配** | 容易的卡片间隔更长，难的卡片频繁出现 |
| **自我解释** | 自评过程迫使用户诚实地评估记忆质量 |

### 阶段 ⑤：强化——错误驱动 × 易混分析

当某个知识点反复出错时，系统不只是增加频率——还要改善「编码质量」。

**错误强化三步走：**

```
① 检测：连续 3 次评分 < 3 → 标记为薄弱

② 分析：为什么薄弱？
   ├─ 单纯遗忘 → 缩短间隔即可
   ├─ 与其他概念混淆 → AI 生成对比材料
   └─ 理解偏差 → AI 生成解释/记忆技巧

③ 干预：
   ├─ 进入重点强化页专项学习
   ├─ 在知识图谱中高亮该节点及关联节点
   └─ 如果是混淆导致 → 触发易混概念对比卡片
```

**渐进式记忆的体现：**

错误强化不是让你「重新学」，而是让你在**已有记忆残留的基础上修正**，这比从零开始学效率高得多。

### 阶段 ⑥：输出——长期记忆 → 考试输出

当卡片进入 60 天以上的间隔，且用户多次自评「完全掌握」：

- 卡片标记为「已掌握」（mastered）

- 领域掌握度统计更新
- 用户可在论述骨架中看到该知识点的实际应用

| 考试题型 | 对应的系统功能 |

# 第四部分：四大核心机制详解

## 4.1 主动回忆（Active Recall）

### 理论依据

> 被动阅读的记忆留存率：1 周后约 10-20%
> 主动回忆的记忆留存率：1 周后约 60-80%

### 系统实现

**1. 强制停顿设计**

卡片正面只显示问题，翻转按钮不是「显示答案」，而是「我已思考，查看答案」。这个措辞差异很关键——它暗示用户必须先思考。

**2. 不可逆翻转**

看到答案后不能返回。这防止了「瞟一眼答案然后说我会了」的自我欺骗。

**3. 必须自评**

没有「跳过」按钮。每一张卡片都必须评分。这确保用户诚实地面对自己的记忆状态。

**4. 无提示**

问题面不提供任何线索、选项、提示。如果是多选题：
- 正面：「下列哪些属于'四个全面'？」
- 无选项展示
- 用户脑中回忆
- 翻转后看到完整选项 + 正确答案 + 解析

### 为什么这样设计

人类的记忆提取有两种模式：
- **识别（Recognition）：** 看到选项后认出正确答案 → 浅层加工，容易遗忘
- **回忆（Recall）：** 没有任何线索下从记忆中提取 → 深层加工，长期留存

主动回忆强制用户进入「回忆」模式，而不是「识别」模式。

## 4.2 艾宾浩斯遗忘曲线

### 理论依据

```
记忆强度
  100% │●
       │  ●
   80% │    ●
       │       ●
   60% │          ●
       │              ●
   40% │                   ●
       │                         ●
   20% │                                ●
       │____________________________________●___
       0    1h   1d   2d   4d   7d   15d  30d
                               时间 →

       每次在遗忘临界点复习 → 遗忘曲线变平缓 →
       间隔可以越来越长 → 最终形成长期记忆
```

### 系统实现

**核心原则：在即将遗忘的时候复习，而不是在还记得的时候重复。**

| 复习时机 | 传统做法 | 本系统 |
|---------|---------|--------|
| 刚学完 | 反复看 | ❌ 不复习（纯属浪费时间） |
| 5 分钟后 | 不管 | ✅ 第一次复习（短期→长期的关键窗口） |
| 30 分钟后 | 不管 | ✅ 第二次复习 |
| 12 小时后 | 不管 | ✅ 第三次复习（睡眠前巩固） |
| 1 天后 | 可能忘了才看 | ✅ 第四次复习 |
| 3 天后 | 忘了 | ✅ 第五次复习 |
| 7 天后 | 忘了 | ✅ 第六次复习 |
| … | | 间隔持续增长 |

**算法不是固定间隔，而是动态调节：**

```
如果质量评分高（≥4）：
  间隔 = 间隔 × easeFactor（增长）
  
如果质量评分低（<3）：
  间隔重置为 1 天
  
如果质量评分中等（3）：
  间隔正常增长但 easeFactor 不变
```

**这意味着每个人的复习节奏都是个性化的。**

## 4.3 错误强化（Error-Driven Reinforcement）

### 理论依据

> 人在犯错后修正的记忆，比从未犯错的记忆更牢固。

### 系统实现

**1. 错误检测（自动）**

```typescript
// 薄弱点判定逻辑
function isWeak(card: MemoryCard): boolean {
  const recentHistory = card.history.slice(-5);
  
  // 条件 1：连续 3 次低评分
  const consecutiveLow = card.history.slice(-3)
    .every(h => h.quality < 3);
  
  // 条件 2：最近 5 次中 ≥ 3 次低评分
  const frequentLow = recentHistory
    .filter(h => h.quality < 3).length >= 3;
  
  // 条件 3：easeFactor 过低
  const lowEase = card.easeFactor < 1.5;
  
  return consecutiveLow || frequentLow || lowEase;
}
```

**2. 错误分析（AI 驱动）**

检测到薄弱后，AI 分析原因：
- 输入：该卡片的历史评分 + 标准答案 + 相关知识点的评分
- 输出：薄弱原因 + 强化建议

**3. 错误干预（渐进式）**

| 干预层级 | 触发条件 | 干预措施 |
|---------|---------|---------|
| L1 | 连续 2 次低评分 | 缩短间隔 50% |
| L2 | 连续 3 次低评分 | 进入重点强化页 + 显示易混对比 |
| L3 | 连续 5 次低评分 | AI 重新生成该卡片的答案表述 + 添加记忆技巧 |

**4. 错误驱动 ≠ 惩罚**

关键设计理念：错误不是用户的问题，是「编码质量」的问题。

错误强化是在改善知识的呈现方式，而不是让用户反复做题。这保护了用户的学习动力。

## 4.4 渐进式记忆（Progressive Memory）

### 理论依据

> 记忆不是 0 或 1，而是一个从「完全陌生」到「自动提取」的连续谱。

### 四个记忆阶段

```
阶段 1：编码期（Encoding）
  "这是什么？"
  首次接触知识点
  学习卡片正面 → 翻转 → 阅读答案
  
阶段 2：巩固期（Consolidation）
  "我好像记得一些……"
  5min → 30min → 12h 短间隔复习
  从短期记忆转移到长期记忆
  
阶段 3：提取期（Retrieval）
  "我知道这个！"
  1d → 3d → 7d 间隔扩大
  回忆越来越轻松，速度越来越快
  
阶段 4：自动化期（Automaticity）
  "不需要想就能说出来"
  30d → 60d → 120d 间隔
  可以在论述题中自动调用
```

### 系统如何支持四个阶段

| 阶段 | 卡片状态 | 间隔范围 | 系统行为 |
|------|---------|---------|---------|
| 编码期 | NEW | - | 首次翻转展示完整答案 + 关键词高亮 + 知识树位置 |
| 巩固期 | LEARNING | 5min-12h | 短间隔密集复习 + 显示「上次间隔：5 分钟前」 |
| 提取期 | REVIEWING | 1d-30d | 正常间隔 + 显示历史正确率 |
| 自动化期 | MASTERED | 60d+ | 低频复习 + 标记为已掌握 + 论述骨架中自动引用 |

### 渐进式体现在三个维度

**维度 1：时间间距渐进**

```
学习 → 5min → 30min → 12h → 1d → 3d → 7d → 15d → 30d → 60d → 120d
       ↑ 短间隔密集（巩固期）    ↑ 间隔逐步拉大（提取期）    ↑ 长间隔（自动化期）
```

**维度 2：难度渐进**

新卡片按知识树层级排序——先学根节点（大概念），再学子节点（细节）。这样用户在学细节时已经有了上层框架。

**维度 3：反馈渐进**

| 阶段 | 反馈内容 |
|------|---------|
| 编码期 | 完整答案 + 关键词高亮 + 知识树位置 |
| 巩固期 | 完整答案 + 上次复习时间 |
| 提取期 | 答案 + 历史正确率 + 间隔信息 |
| 自动化期 | 仅展示关键词（最简反馈，确认回忆正确即可） |

# 附录 A：核心名词定义

| 名词 | 定义 |
|------|------|
| 知识领域（Domain） | 党课知识的顶层分类，如「习近平新时代中国特色社会主义思想」 |
| 知识点（Knowledge Point） | 一个独立的知识单元，可转化为一张或多张卡片 |
| 记忆卡片（Memory Card） | 一张可翻转的问题-答案对，是记忆系统的最小交互单元 |
| 编码（Encoding） | 将原始文本转化为结构化知识点和卡片的过程 |
| 自评（Self-Assessment） | 用户在翻转卡片后对自己回忆质量的 0-5 评分 |
| 间隔（Interval） | 两次复习之间的时间距离（天） |
| 舒适度（Ease Factor） | SM-2 算法的核心参数，控制间隔增长速度 |
| 薄弱点（Weak Point） | 连续多次评分低的知识点，需要额外强化 |
| 易混概念（Confusing Pair） | AI 识别的两个容易混淆的知识点 |
| 论述骨架（Essay Skeleton） | AI 生成的论述题答题框架（总-分-总结构） |
| 掌握（Mastered） | ease >= 2.5 且 repetitions >= 5，进入低频复习 |

# 附录 B：AI Prompt 设计原则

1. **每个 AI 功能有且只有一个职责。** 不写万能 Prompt。
2. **Prompt 输出必须是固定 JSON Schema。** 用 Zod 校验，解析失败则重试（最多 2 次）。
3. **温度设置：** 知识提炼 0.1（需要精确），判分 0.2，易混识别 0.4（需要创造力），骨架生成 0.5。
4. **所有 Prompt 包含「宁可多，不可漏」原则。**
5. **所有 Prompt 包含党课术语的严格性要求：** 不得篡改、简化、曲解原文中的政治表述。
6. **每次 AI 调用记录日志：** model, tokens, latency, success/fail（用于后续成本优化）。
