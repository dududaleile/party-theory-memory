## 文档定位

本文档是系统的**核心逻辑总纲**。后续所有开发以此文档为唯一真实来源。

## 设计约束（不可违背）

### 约束 1：知识逻辑优先于页面

```
页面只是知识的载体。
重点永远是「知识如何形成体系」。

禁止：知识点平铺成背诵列表
禁止：做成刷题工具
禁止：无层级、无关联的碎片记忆

必须：整体 → 局部 → 细节
必须：用户脑中形成结构化知识体系
```

### 约束 2：记忆训练优先级

```
这是党课学习的特殊性决定的——关键词的准确复述是考试得分核心。

训练优先级（从高到低）：

① 主动回忆
   用户面对问题，脑中独立回忆答案
   任何记忆训练的起点
   不可跳过、不可降级

② 关键词检测
   回忆的答案中，核心政治术语是否准确？
   「以人民为中心」不能写成「以人为本」
   「全面从严治党」不能写成「加强党的建设」

③ 默写 / 完整输出
   对于论述题考点，能否结构化地输出完整答案？
   不仅是记住关键词，还要能组织成段落

④ 论述骨架
   大题答题框架（总-分-总）
   这是输出的最高形式，需要在①②③的基础上

不是刷选择题。
不做选择题题库。
```

### 约束 3：知识导入后自动建体系

```
导入知识后，系统必须自动建立：

  主题（Topic）         → 知识点的顶层归类
  上级知识（Parent）     → 该知识点属于哪个更大的概念
  下级知识（Children）   → 该知识点包含哪些子概念
  关联知识（Related）    → 语义相关但不在同一分支的知识点
  易混知识（Confusing）  → 容易混淆的知识点对

最终目标：
  用户脑中形成的是结构化的知识体系
  不是一堆孤立的事实
```

# 第一部分：知识结构模型

## 1.1 知识组织方式

```
禁止的（平铺）：
  · 知识点 1
  · 知识点 2
  · 知识点 3
  ...
  → 用户脑中：一堆碎片

必须的（体系化）：
  习近平新时代中国特色社会主义思想
  ├── 核心要义
  │   ├── 坚持和发展中国特色社会主义
  │   │   ├── 总任务
  │   │   ├── 总布局（五位一体）
  │   │   └── 战略布局（四个全面）
  │   ├── 实现中华民族伟大复兴
  │   └── 以人民为中心的发展思想
  ├── "八个明确"
  │   ├── 明确一
  │   └── ...
  └── "十四个坚持"
      ├── 坚持一
      └── ...

  → 用户脑中：一棵有逻辑的知识树
  → 可以从任何一个节点向上追溯、向下展开、横向关联
```

## 1.2 五大知识关系

```
知识点导入后自动建立的五种关系：

① 主题关系（Topic）
  知识点 → 知识领域 → 知识大类
  例如：「四个全面」→「习近平新时代中国特色社会主义思想」→「党的理论」

② 上下级关系（Parent-Child）
  父知识点 → 子知识点
  例如：「四个全面」→「全面从严治党」
  确保：先学父概念，再学子细节

③ 下级展开关系（Children）
  一个知识点包含哪些子知识点
  例如：「八项规定」展开为：
    - 改进调查研究
    - 精简会议活动
    - 精简文件简报
    - 规范出访活动
    - 改进警卫工作
    - 改进新闻报道
    - 严格文稿发表
    - 厉行勤俭节约

④ 关联关系（Related）
  不同分支中语义相关的知识点
  例如：「以人民为中心」↔「群众路线」↔「人民当家作主」
  它们不在同一分支，但逻辑关联

⑤ 易混关系（Confusing）
  容易混淆的知识点对
  例如：「群众路线 ⚡ 群众观点」
  一个是方法论（怎么走），一个是世界观（怎么看）
```

## 1.3 知识树的构建规则

```
规则 1：有「整体」才有「局部」
  必须先存在父知识点，子知识点才能挂载
  学习时先学父，后学子

规则 2：一个知识点只有一个父节点
  向上追溯路径唯一
  树结构清晰

规则 3：关联关系通过独立的「边」表达
  不破坏树结构
  关联边可有多个，无限制

规则 4：每个知识点知道自己完整的树路径
  treePath = [根领域, ..., 直接父节点]
  例如：["xi-thought", "eight-clears", "clear-1"]
  
规则 5：知识点的顺序有意义
  同级知识点按 order 字段排序
  学习顺序 = 树深度升序 + order 升序
```

# 第二部分：记忆训练体系

## 2.1 训练层级（由核心到外围）

```
层级 1：主动回忆（Active Recall）
  │  操作：看到问题 → 脑中独立回忆 → 翻转查看答案
  │  覆盖：100% 的卡片
  │  这是不可跳过的核心层
  │
  ▼
层级 2：关键词检测（Keyword Verification）
  │  操作：对比自己的回忆与标准答案中的关键词
  │  系统：答案中的关键词自动高亮
  │  用户：逐词确认自己是否准确复述
  │  覆盖：100% 的卡片（所有卡片都有 keywords）
  │
  ▼
层级 3：完整输出（Full Output）
  │  操作：对论述题考点，输入自己的完整答案
  │  系统：AI 比对用户答案与标准答案
  │  反馈：哪些要点覆盖了、哪些遗漏了
  │  覆盖：questionType = "essay" 的卡片
  │
  ▼
层级 4：论述骨架（Essay Skeleton）
  │  操作：学习 AI 生成的总-分-总答题框架
  │  用途：考试前结构化复习
  │  覆盖：用户标记为「论述题考点」的知识点
```

## 2.2 关键词检测的具体机制

```
每一张卡片翻转后：

1. 答案中的关键词自动高亮（暖黄背景 #FFF3CD）
   例如：
   「全面建设社会主义现代化国家、全面深化改革、
     全面依法治国、全面从严治党」

2. 底部显示关键词标签列表
   📎 四个全面 | 战略布局 | 现代化 | 深化改革 | 依法治国 | 从严治党

3. 用户对比自己的回忆：
   □ 「四个全面」 — 我说对了吗？
   □ 「战略布局」 — 我说对了吗？
   □ 「现代化」   — 我说对了吗？
   ...

4. 然后才进行 4 级自评

为什么关键词检测在自评之前？
  因为党课记忆的核心是「精确复述关键术语」。
  用户可能记得大意但术语不精确，这在考试中会失分。
  关键词检测让用户精准地知道自己哪里不精确。
```

## 2.3 不做的事

```
不做选择题题库。
  理由：选择题训练的是「识别」而非「回忆」。
        识别能力的记忆留存率远低于回忆能力。
        党课考试中，选择题只占一部分，简答和论述才是大头。

不做自动播放/闪卡。
  理由：被动观看 = 被动遗忘。必须主动回忆。

不做排行榜/积分/游戏化。
  理由：党课学习不需要外部激励。形成结构化知识体系本身就是激励。
```

## 第四部分：知识呈现层级

## 4.1 三层递进（不可跳级）

```
层级 1：整体（领域概览）
  │
  │  呈现位置：首页
  │  呈现内容：知识领域列表 + 环形掌握度
  │  用户行为：点击领域 → 进入该领域
  │  目的：建立「知识版图」
  │
  ▼
层级 2：局部（领域内结构）
  │
  │  呈现位置：领域详情 / 知识树视图
  │  呈现内容：该领域的子领域 + 直系知识点
  │  用户行为：点击知识点 → 开始学习
  │  目的：建立「领域内有什么」的认知
  │
  ▼
层级 3：细节（单点聚焦）
  │
  │  呈现位置：卡片学习页
  │  呈现内容：一个问题 + 面包屑路径
  │  用户行为：主动回忆 → 翻转 → 关键词检测 → 自评
  │  目的：精准记忆一个知识点
  │
  ▼
跨层级补充：知识图谱（全局关联）
  │
  │  呈现位置：重点强化页
  │  呈现内容：力导向图
  │  用户行为：缩放、拖拽、点击查看
  │  目的：看到知识点之间的关联网络
```

## 4.2 面包屑导航 — 关键设计

```
学习/复习卡片顶部的面包屑：

  习近平新时代 › 核心要义 › "八个明确" › 第一条

  作用：
    ① 每张卡片都告诉用户「你在这里」
    ② 点击任一层级 → 返回该层级列表
    ③ 记忆时关联位置（空间记忆辅助语义记忆）

  样式：
    字号 11pt、三级文字色
    层级分隔符 ›（精致）
    当前层级颜色加深
    不抢卡片焦点
```

## 第五部分：数据保证知识体系

## 5.1 知识点表的关键字段

```
knowledgePoints 表：

  domainId          → 所属领域（主题关系）
  parentPointId     → 父知识点（上下级关系）
  treePath          → 完整路径数组（冗余，避免递归查询）
  keywords          → 关键词列表（用于关键词检测 + 覆盖度计算）
  questionType      → 题型标记（决定训练层级）
```

## 5.2 知识关联表

```
knowledgeRelations 表：

  sourcePointId     → 源知识点
  targetPointId     → 目标知识点
  relationType      → 关系类型
    "parent_child"   — 树中的父子关系
    "prerequisite"   — 前置关系（学 A 之前需要学 B）
    "related"        — 一般语义关联
    "contradicts"    — 易混淆（辨析关系）
    "extends"        — B 是 A 的深化/扩展
  strength          → 关系强度 0-1
```

## 5.3 MemoryCard 表的关键字段

```
memoryCards 表：

  pointId           → 关联的知识点
  stability (S)     → 记忆稳定性
  difficulty (D)    → 卡片固有难度
  memoryStrength    → 五维度综合评分 0-100
  isWeak            → 是否薄弱点
  learningPhase     → 编码/巩固/提取/掌握
  nextReview        → 下次复习时间戳
  recentRatings     → 最近 5 次评分
```

## 设计约束检查清单

```
在后续所有开发中，每一项功能必须通过以下检查：

□ 这是帮助用户建立知识体系，还是让用户死记硬背？
□ 知识点在这里是否有逻辑层级？还是平铺？
□ 用户能否感知到当前知识点在整体知识树中的位置？
□ 关键词是否被高亮和检测？
□ 记忆训练是否以主动回忆为起点？
□ 有没有任何形式的「被动浏览」替代了「主动回忆」？
□ UI 是否在帮助知识呈现，还是在抢知识的焦点？
□ 导入的知识是否自动建立了主题/上下级/关联/易混关系？
□ 新卡片的学习顺序是否遵循树深度升序？
□ 有没有任何功能在鼓励「刷题」而非「建立体系」？
```
